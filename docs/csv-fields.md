# SEO 爬蟲 CSV 報表欄位說明

這份 CSV 報表是用來快速檢查網站頁面的基礎 SEO 狀態。每一列代表一個被爬蟲檢查到的頁面，每一欄代表該頁面的某個 SEO 或技術狀態指標。

## 欄位說明

| 欄位名稱 | 中文名稱 | 代表意涵 | SEO 分析重點 |
| --- | --- | --- | --- |
| `url` | 頁面網址 | 被爬蟲實際檢查的頁面網址。 | 用來確認哪些頁面已被納入檢查，也可作為後續修正問題的定位依據。 |
| `status_code` | HTTP 狀態碼 | 伺服器回應此頁面的狀態，例如 `200`、`301`、`404`。 | `200` 代表頁面正常；`404` 代表頁面不存在；`301/302` 代表轉址。大量錯誤頁會影響搜尋引擎爬取效率與使用者體驗。 |
| `title` | 頁面標題 | HTML `<title>` 的內容，通常會顯示在搜尋結果標題與瀏覽器分頁。 | 每個重要頁面都應該有清楚且獨特的 title，並包含主要關鍵字。 |
| `title_length` | 頁面標題長度 | `title` 的字元數。 | 標題太短可能資訊不足；太長可能在搜尋結果中被截斷。可用來篩選需要優化的頁面。 |
| `meta_description` | 頁面描述 | HTML `<meta name="description">` 的內容，常被搜尋引擎用作搜尋結果摘要。 | 好的描述能提升搜尋結果點擊率。缺少或重複的描述通常需要補強。 |
| `meta_description_length` | 頁面描述長度 | `meta_description` 的字元數。 | 描述太短可能吸引力不足；太長可能被搜尋結果截斷。 |
| `h1_count` | H1 標題數量 | 頁面中的 `<h1>` 數量。 | H1 通常代表頁面主標題。一般建議重要頁面至少有一個清楚的 H1，且避免過多 H1 造成主題不明確。 |
| `h1_text` | H1 標題文字 | 頁面中所有 H1 的文字內容，多個 H1 會用 `|` 分隔。 | 可檢查頁面主題是否明確、是否包含核心關鍵字，以及是否與 title 對應。 |
| `canonical` | 標準網址 | HTML `<link rel="canonical">` 指定的標準頁面網址。 | 用來告訴搜尋引擎哪個 URL 是主要版本，可避免重複內容分散 SEO 權重。缺少或設定錯誤時需特別檢查。 |
| `image_count` | 圖片數量 | 頁面中的圖片總數。 | 圖片多的頁面要注意圖片壓縮、載入速度與替代文字。 |
| `images_missing_alt` | 缺少 alt 的圖片數量 | 沒有 `alt` 屬性，或 `alt` 內容為空的圖片數量。 | alt 文字有助於圖片 SEO、無障礙閱讀與搜尋引擎理解圖片內容。數量越高，代表圖片 SEO 還有優化空間。 |
| `images_missing_alt_ratio` | 缺少 alt 的圖片比例 | `images_missing_alt` 除以 `image_count` 的比例。 | 可用來快速判斷該頁圖片替代文字的完整度。比例越高，代表越需要補充圖片 alt。 |
| `internal_links_count` | 內部連結數 | 頁面中連到同一網站網域內的連結數量。 | 內部連結有助於搜尋引擎探索網站內容，也能傳遞頁面權重。過少可能代表頁面孤立，過多則需檢查是否雜亂。 |
| `external_links_count` | 外部連結數 | 頁面中連到其他網站的連結數量。 | 外部連結可提供補充資料或合作來源，但過多或品質不佳的外部連結可能影響使用者體驗與信任度。 |
| `error` | 錯誤訊息 | 爬蟲抓取或分析頁面時遇到的錯誤。 | 若此欄有內容，代表該頁可能無法正常檢查。需要先排除連線、權限、非 HTML 內容或伺服器錯誤等問題。 |

## 建議的分析順序

1. 先看 `status_code`，找出 `404`、`500` 或異常轉址頁面。
2. 檢查 `title`、`meta_description` 是否缺少、過短、過長或重複。
3. 檢查 `h1_count` 是否為 `0`，或是否出現過多 H1。
4. 檢查 `canonical` 是否存在，且是否指向正確頁面。
5. 依 `images_missing_alt_ratio` 找出圖片 alt 缺漏比例高的頁面。
6. 檢查 `internal_links_count` 過低的頁面，確認是否需要增加內部連結。
7. 最後檢查 `error` 欄位，確認是否有頁面沒有被完整分析。

## 給業主的簡短解讀

這份報表可以協助我們找出網站在搜尋引擎可讀性、頁面標題與描述、圖片替代文字、內部連結結構，以及技術可存取性上的基礎問題。第一階段建議先處理「錯誤頁面」、「缺少 title / description / H1」、「圖片缺少 alt」與「canonical 設定異常」這幾類問題，因為這些通常是最容易發現、也最容易優先改善的 SEO 項目。
